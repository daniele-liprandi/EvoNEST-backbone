## Getting Started

### Prerequisites

- Docker and Docker Compose
- Node.js (for local development)
- MongoDB (for local development)

### Setting Up Your Development Environment

1. Fork the repository on GitHub.
2. Clone your fork locally:
   ```
   git clone https://github.com/your-username/evonest.git
   cd evonest
   ```
3. Install dependencies:
   ```
   npm install
   ```
4. Create a branch for your feature or bug fix:
   ```
   git checkout -b feature/your-feature-name
   ```

## How to Submit Changes

1. Make your changes in your feature branch.
2. Add or update tests as necessary.
3. Ensure all tests pass:
   ```
   npm test
   ```
4. Commit your changes:
   ```
   git commit -m "Brief description of your changes"
   ```
5. Push to your fork:
   ```
   git push origin feature/your-feature-name
   ```
6. Submit a pull request to the `main` branch of the EvoNEST repository.

## Coding Conventions and Style Guidelines

- We use ESLint for JavaScript linting. Please ensure your code adheres to our ESLint configuration.
- Follow the [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript).
- Use meaningful variable and function names.
- Write clear, concise comments and documentation.
- Keep functions small and focused on a single task.
- Use TypeScript for type safety where applicable.

## How to Report Bugs

If you find a bug, please submit an issue. When filing an issue, make sure to answer these questions:

1. What version of EvoNEST are you using?
2. What operating system and processor architecture are you using?
3. What did you do?
4. What did you expect to see?
5. What did you see instead?

## How to Suggest Enhancements

If you have an idea for a new feature or an enhancement to existing functionality, we'd love to hear about it! Please submit an issue with the following information:

1. A clear and descriptive title for the issue.
2. A detailed description of the proposed enhancement.
3. An explanation of why this enhancement would be useful to most EvoNEST users.
4. Any relevant examples or mock-ups if applicable.
